# 关键数字

当一页的重点是几个数字或结论时，用这个模板会更稳。

- **建议时长** `3 天`
  节奏比较完整。
- **重点区域** `3 条线`
  老城、兵马俑、博物馆。
- **最佳时段** `傍晚`
  夜景和气温都更友好。
- **提前准备** `预约`
  热门馆更依赖预约。

::right::

## 资料来源

数字可以来自官方开放时间、交通估算或内部项目数据。

## 使用建议

左边只放真正重要的数字，右边负责解释这些数字意味着什么。

# 关键数字（稀疏版）

这页用来测试左侧只有两张数字卡时，卡片和右侧说明是否还能平衡。

- **建议时长** `2 天`
  只能抓最核心的第一轮体验。
- **最远通勤** `1 次`
  兵马俑那天尽量单独安排。

::right::

## 怎么理解

当数字更少时，版面应该保留留白，而不是把两个块硬撑满。

## 适合什么内容

适合特别明确、特别有限的两到三个判断点。

# 关键数字（密集版）

这页用来测试左侧六张数字卡、右侧三组说明同时存在时的密度上限。

- **建议时长** `3 天`
  第一次去最稳。
- **远途天数** `1 天`
  兵马俑最好单独安排。
- **老城主线** `1 条`
  钟楼、鼓楼、城墙可以串起来。
- **慢逛傍晚** `2 次`
  至少留两段给夜景和散步。
- **预约项目** `2 个`
  热门馆和部分演出更依赖提前锁定。
- **换酒店次数** `0`
  三天行程最好住同一片区。

::right::

## 数据来源

可以来自开放时间、路程估算、订票规则，或者作者自己的实地经验。

## 版面建议

左边数字越多，单张卡片就越应该更短，不要把解释文字也全部塞进去。

## 适合场景

适合一页先把判断讲透，再在后面页展开解释。

# 数字卡也要能带图

这页用来测试左侧指标卡内部插入 Markdown 图片时，卡片是否还能保持统一节奏。

- **夜游热度** `78%`
  热门时段主要集中在傍晚后的两小时。

  ![夜间街区人流](https://images.unsplash.com/photo-1507525428034-b723cf961d3e?auto=format&fit=crop&w=1400&q=80)
- **内容停留** `42 分钟`
  真正慢看的内容通常需要单独留出一段。
- **预约依赖** `2 项`
  热门馆和高峰时段都更依赖提前确认。

::right::

## 图片进入卡片

只要还是 Markdown，这种 media card 也应该能稳定工作。

## 这页在测什么

它主要看列表卡片里的图片比例、卡片高度和右侧说明是否还能保持平衡。
