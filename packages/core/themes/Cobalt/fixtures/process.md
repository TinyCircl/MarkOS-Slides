# 执行步骤

当左边天然是顺序动作时，这个模板最顺手。

1. **确定范围**
   先确认这页到底是在讲节奏、地点还是建议。
2. **压出层级**
   把最重要的标题和说明先分出来。
3. **安排主次**
   左边负责主视觉或主结构，右边负责解释。
4. **收束文案**
   最后再修字句，不要一开始就陷进微调。

::right::

## 适合什么内容

适合步骤、方法、行程顺序和带编号的执行流程。

## 不适合什么内容

如果左边不是天然顺序，而只是普通列表，就不要硬套这个模板。

# 执行步骤（稀疏版）

这页用来测试只有三步时，左侧是否还能保持稳定节奏。

1. **先定主轴**
   先决定这一页讲的是节奏、区域还是选择。
2. **再分左右**
   左边放主结构，右边补解释，不要两边都抢主角。
3. **最后压句子**
   样式先稳住，再回头改文案。

::right::

## 适合什么内容

适合很清楚的三步法、三段式流程，或者一天行程的三段推进。

## 这页在测什么

步骤更少时，版面是否还能显得完整，而不是像缺内容。

# 执行步骤：从素材收集到最终上线（多人协作版）

这页用来测试长标题和六步流程同时出现时，模板的密度上限。

1. **收原始内容**
   先把文字、数据、素材和链接收齐。
2. **确认页角色**
   决定这页是总览、对比、流程还是数字。
3. **拆出层级**
   把主标题、辅标题和注释分开。
4. **挑模板**
   先选结构，再做视觉微调。
5. **看密度**
   判断是稀疏、标准还是偏满。
6. **最后验收**
   回到真实渲染里看每页是否稳定。

::right::

## 适合什么内容

适合步骤、方法论、流程拆解，以及确实带有顺序关系的行程页。

## 容易出错的地方

如果左边步骤过长，应该先缩句子，而不是先怪模板扛不住。

## 这页在测什么

它主要用来观察标题换行、步骤卡高度和左右列在高密度下的平衡。
